正则表达式

发表于 2021-03-30 更新于 2021-04-10 分类于 学习手册

定义

正则表达式是用于匹配字符串中字符组合的模式

简单的正则表达式

使用特殊字符

断言

断言的组成之一是边界。对于文本、词或模式，边界可以用来表明它们的起始或终止部分

边界类断言
  • ^ : 匹配输入的开头
  • $ : 匹配输入的结束
  • \b : 匹配一个单词的边界
  • \B : 匹配非单词边界
其他断言
  • x(?=y) : 向前断言，x 被 y 跟随
  • x(?!y) : 向前否定断言，x不被y跟随
  • (?<=y)x : 向后断言，x跟随y
  • (?<!y)x : 向后否定断言，x不跟随y

字符类

常用
  • . : 匹配除行终止符之外的任何单个字符， \n，\r，\u2028，\u2029
  • \d : 匹配任意数字，[0-9]
  • \D : 匹配任意非数字
  • \w : 匹配基本拉丁字母中的任何字母数字字符，包括下划线，[A-Za-z0-9_]
  • \W: 匹配任何不是来自基本拉丁字母的单词字符
  • \s : 匹配一个空白字符，包括空格、制表符、换页符和换行符，[ \f\n\r\t\v\u00a0\u1680\u180e\u2000-\u200a\u2028\u2029\u202f\u205f\u3000\ufeff]
  • \S : 匹配一个非空白字符
不常用
  • \t : 匹配一个水平制表符
  • \v : 匹配一个垂直制表符
  • \r : 匹配一个回车符
  • \n : 匹配一个换行符
  • \f : 匹配一个换页符
  • [\b] : 匹配一个退格
  • \0 : 匹配 NULL（U+0000）字符
  • \cX : 当X是处于A到Z之间的字符的时候，匹配字符串中的一个控制符
  • \xhh : 匹配一个两位十六进制数（\x00-\xFF）表示的字符
  • \uhhhh : 匹配一个四位十六进制数表示的 UTF-16 代码单元
  • \u{hhhh} or \u{hhhhh} : （仅当设置了u标志时）匹配一个十六进制数表示的 Unicode 字符

组和范围

  • x|y : 匹配 “x” 或 “y” 任意一个字符
  • [xyz] or [a-c] : 字符集。 匹配任何一个包含的字符
  • [^xyz] or [^a-c] : 一个否定的或被补充的字符集。匹配任何没有包含在括号中的字符
  • (x) : 捕获组。匹配x并记住匹配项
  • \n : 引用第n个捕获组
  • (?<name>x) : 具名捕获组。匹配”x”并将其存储在返回的匹配项的groups属性中
  • (?:x) : 非捕获组。匹配 “x”，但不记住匹配项(应用场景： 较多括号用于分组)

量词

  • x* : 匹配“x”0次或多次
  • x+ : 匹配“x”1次或多次
  • x? : 匹配“x”0或1次
  • x{n} : 匹配“x”n次
  • x{n,} : 匹配“x”至少n次
  • x{n, m} : 匹配“x”n-m次（n<m && n >= 0）
  • ? : 非贪婪匹配，一旦找到匹配就停止

通过标志进行高级搜索

格式

标志

  • g : 全局搜索
  • i : 不区分大小写搜索
  • m : 多行搜索
  • s : 允许.匹配换行符
  • u : 使用unicode码的模式进行匹配
  • y : 执行“粘性(sticky)”搜索,匹配从目标字符串的当前位置开始

正则表达式使用方法

  • exec : 在字符串中执行查找匹配，返回一个数组（未匹配到则返回 null）
  • test : 在字符串中测试是否匹配，返回 true 或 false
  • match : 在字符串中执行查找匹配，返回一个数组，在未匹配到时会返回 null
  • matchAll : 在字符串中执行查找所有匹配，返回一个迭代器（iterator）
  • search : 在字符串中测试匹配，返回匹配到的位置索引，或者在失败时返回-1
  • replace : 在字符串中执行查找匹配，并使用替换字符串替换掉匹配到的子字符串
  • split : 使用正则表达式或者一个固定字符串分隔一个字符串，并将分隔后的子字符串存储到数组中
